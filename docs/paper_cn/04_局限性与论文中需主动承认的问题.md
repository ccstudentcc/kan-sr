# 局限性与论文中需主动承认的问题

## 本页可直接回指的证据文件

- [`../../EXPERIMENT_PROTOCOL.md`](../../EXPERIMENT_PROTOCOL.md)：本页关于预算字段、实验协议与执行边界的主要来源。
- [`../../CURRENT_WORK_REFERENCE.md`](../../CURRENT_WORK_REFERENCE.md)：本页关于继承资产与历史上下文的辅助来源。
- [`../../output/summary/univariate_quadratic.csv`](../../output/summary/univariate_quadratic.csv) 与 [`../../output/summary/multivariate_sinexp.csv`](../../output/summary/multivariate_sinexp.csv)：本页关于当前统一链路结果规模的直接来源。
- [`../../notebooks/Comparison_of_different_methods_for_SR_multivariate.ipynb`](../../notebooks/Comparison_of_different_methods_for_SR_multivariate.ipynb)：本页关于 notebook 与统一脚本不能混写的直接来源。

## 1. 统计稳健性仍然不足

当前统一对比结果里的 `n_repeats` 仍然是 `1`。这意味着：

- 结果可以作为阶段性对比证据使用；
- 但不宜写成“统计结论已经稳定成立”；
- 更不适合做强形式的显著性叙述。

## 2. notebook 与统一脚本结果不能混成同一批证据

多变量任务曾出现过 notebook 目标与脚本目标不一致的问题。当前 `output/summary` 中的结果应视为修复后的统一口径，而 notebook 旧输出只能作为历史上下文，不应和当前 summary 混写。

## 3. 预算公平性需要谨慎表述

`EXPERIMENT_PROTOCOL.md` 已说明 `time_budget_seconds` 目前主要还是计划与审计字段，而不是强制超时中断机制。因此：

- 可以写“统一记录了预算字段”；
- 不能写“所有方法都已在严格统一时间控制下运行”。

## 4. 指标维度仍偏有限

当前统一链路里更成熟的指标主要是：

- `MSE`
- `R2`
- `time_seconds`
- `complexity`

噪声鲁棒性、外推能力、失败模式和更强的统计诊断还没有完整补齐。

## 5. 表达式可读性并不总是成熟

单变量任务可以直接展示四种方法的 raw 表达式；但多变量任务中的 `kan` 结果目前仍是数值模型标记，而不是清晰可读的符号公式。因此论文在写到“公式恢复”时需要收敛语气。

## 6. 历史资产与新增工作必须分开写

历史 `*.pkl` 与 `model/` 目录中的状态快照不能被直接写成“本研究本轮新增训练结果”。更稳妥的写法是：

- 这些资产构成了项目已有基础；
- 本轮新增工作更多体现在协议统一、结果汇总和材料整理上。
