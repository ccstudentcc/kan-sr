## 1. 特殊函数实验：正弦积分

来源：`notebooks/Special_function.ipynb`

### 当前可直接引用的结果

- PyKAN（`width=[1,2,1,1]`）
  - 符号化前：`MSE = 1.555023 × 10^-3`
  - 符号化后：`MSE = 1.440282 × 10^-3`
- EfficientKAN（`width=[1,2,1,1]`）
  - 数值模型：`MSE = 4.035907 × 10^-5`
- Complex PyKAN（`width=[1,5,2,1]`）
  - 符号化前：`MSE = 1.611167 × 10^-5`
  - 符号化后：`MSE = 1.088930 × 10^-4`

### 当前更稳妥的解释

1. “符号化是否改善效果”与网络结构相关，而不是统一成立。
2. 基础 PyKAN 上符号化后误差略降，说明符号约束可能带来轻微增益。
3. 更复杂结构上，符号化后反而可能引入精度回退。
4. EfficientKAN 在结构简洁性与数值表现之间仍具竞争力。

## 2. 无监督关系发现

来源：`notebooks/Unsupervised_Learning.ipynb`

### 任务关系

- `x_2 = sin(6x_0) + exp(2x_1)`
- `x_6 = 4x_3 + x_4 + x_5`

### 更适合写进论文的点

1. 该实验强调的是“仅凭变量观测值恢复结构关系”的能力。
2. notebook 还显式说明了利用先验“不同关系变量不重叠”来逐条恢复关系，这更像方法提示而不是大规模统计结论。
3. 它适合作为结构发现能力的案例，而不是统一基准结果。

## 3. 查找表压缩与奇异性敏感性

来源：`notebooks/Reducing_the_lookup_table_size.ipynb`

### 当前可直接引用的结论

1. 在全部 `d_min` 上，EfficientKAN 的误差最低，MultKAN 与 PyKAN 几乎重合。
2. 当 `d_min = 0.10` 时，EfficientKAN 相比 MultKAN 的误差下降约 `95.97%`（`1.604` vs `39.825`）。
3. 当 `d_min = 0.02` 时，EfficientKAN 仍最优，但相对优势收窄为 `73.75%`（`865.537` vs `3297.257`）。
4. 该实验更支持“奇异性主导难度增长”，而不是“某个模型消除了奇异性”。

### 写作边界

- 这组实验只在截断域内成立。
- 更稳妥的写法是强调局部逼近与数值稳定性，而不是宣称从根本上解决了奇异性问题。
